# Director Decision Record DDR-20260611-011

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
for the source-spec-first HTML explainer UI cleanup and Mermaid render repair.
The task removes unwanted visible boilerplate while preserving source metadata,
structural source evidence, human-only generated-derivative status, and claim
boundaries as non-promotional metadata.

## Accepted Design Decisions

- Reader-facing page titles drop the word `Explainer`; filenames, object IDs,
  registries, and technical explainer terms remain unchanged.
- `source_materials_section` becomes the required visible source-evidence
  control for all tracked explainers.
- `source_drilldowns` and `claim_boundary_toggle` become optional legacy
  controls; they remain valid when declared but are not universal requirements.
- The old visible source drilldown and claim-boundary panels are removed from
  generated HTML. Source paths remain visible in an `All Source Materials`
  section and claim boundaries remain in source-spec metadata.
- Mermaid SVG output is normalized to explicit numeric dimensions derived from
  `viewBox`; the browser fit logic sets canvas and SVG dimensions before
  applying transforms so diagrams render visibly.
- Validators remain structural and deterministic; rendered visual geometry is
  verified through browser QA.

## Claim Boundary

- Allowed: update HTML explainer skill guidance, Documentation Curator role
  guidance, memory bootstrap HTML validator hooks, focused tests, registered
  design documentation, README explainer wording, Markdown HTML explainer
  specs, generated human-only HTML derivatives, governed Mermaid inline-SVG
  renderer behavior, task-local control records, and required registries.
- Forbidden: canonical ontology edits, manuscript TeX edits, PDF edits,
  generated wiki hand edits, generated HTML as independent authority, physics
  claim promotion, Gate Chair verdicts, benchmark promotion, public deployment,
  autonomous proof claims, direct HTML-only edits, claim-boundary meaning
  changes, or registry semantics changes outside task-local control evidence.
