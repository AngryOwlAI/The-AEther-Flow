## Accepted Design Decisions

- The tracked audit command is
  `scripts/project_control/audit_documentation_surfaces.py`.
- The audit checks registered Markdown hashes, HTML explainer source-basis
  links, TeX/PDF derivative links, generated wiki/file/semantic/vault
  registry paths, GitHub-facing Markdown mirror parity, and stale
  documentation-path references.
- Full audit mode checks ignored `.local/` retrieval and vault paths.
- `--skip-local` provides a project-control and CI-safe mode for environments
  where ignored `.local/` surfaces have not been generated.
- `make validate-project-control` uses the CI-safe mode so local retrieval
  state does not become a hidden prerequisite for core project-control checks.

## Claim Boundary

- Allowed: add deterministic documentation-surface audit tooling, add focused
  tests, document the command, wire the CI-safe audit into project-control
  validation, refresh generated memory/wiki registry surfaces, and record the
  project-system receipt.
- Forbidden: canonical ontology TeX edits, manuscript TeX edits, generated HTML
  edits, generated wiki hand edits, PDF edits, physics claim promotion,
  benchmark promotion, Gate Chair verdicts, treating generated artifacts as
  independent authority, or using model judgment as a validator substitute.
