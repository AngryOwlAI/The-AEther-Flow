# Director Decision Record DDR-20260612-005

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
to implement the full HTML explainer upgrade kit at
`implementations_plans/aether_html_explainer_upgrade_kit/`.

The objective is to turn the tracked HTML explainer set from structurally valid
pages with shallow renderer-instruction blocks into deeper source-backed human
documentation while preserving source-spec authority, generated HTML
derivative status, existing Mermaid diagram definitions, and the
exact-GR/open-derivation claim boundary.

## Accepted Design Decisions

- Tracked HTML remains generated from Markdown source specs under
  `markdown/html-explainer-specs/`.
- The upgrade is source-spec first: specifications are updated before tracked
  HTML derivatives are regenerated.
- The kit's depth contract is adopted as project-control design guidance and
  forward operational guidance in the explainer skills.
- A warning-oriented depth lint is added for the observed failure mode:
  content blocks that are shallow or still read like renderer instructions.
- The ontology explainer receives the kit's enhanced treatment: plain-language
  orientation, term cards, quote gallery, adoption-vs-derivation bridge,
  derivation-burden checklist, and explicit non-claim boundaries.
- All ten tracked explainer pages receive page-specific documentation-depth
  upgrades according to the kit's page-by-page plan.
- A shared no-network reader layer is applied to the generated HTML set:
  reading progress, local search, active section navigation, simple/technical
  modes, expand/collapse controls, and copyable source chips.
- Existing Mermaid diagram definitions and inline SVG parity are preserved.
- No runtime Mermaid import, public deployment, or generated-HTML authority
  promotion is introduced.
