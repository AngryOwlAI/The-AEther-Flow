# Director Decision Record DDR-20260612-002

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
for the source-backed summary correction across all currently tracked HTML
explainers. The task supersedes the prior reader-orientation presentation from
`RT-20260612-001` without rewriting historical task records, replacing the
visible `What This Explainer Describes` section with a single prose
`Summary of [Subject]` block.

## Accepted Design Decisions

- Keep `subject_summary` as the internal universal first content-block ID.
- Generated HTML must render `subject_summary` immediately after the
  hero/title area and before the section table of contents.
- The visible section heading must be `Summary of [Subject]`, where `[Subject]`
  is the page title or accepted subject name.
- The visible summary must be one coherent prose block rather than four
  visible facet cards.
- `subject_summary` must contain `data-summary-field` markers for
  `summary_text` and `source_basis`.
- The `source_basis` field must contain visible source-path chips using paths
  declared in the source spec's `source_materials`.
- Source specs must include manually authored source-backed summary prose for
  each tracked page.
- Old visible labels `Reader orientation` and
  `What This Explainer Describes` are obsolete for active tracked HTML pages.
- Documentation Curator ownership is updated through a new
  `documentation-curator@0.4.0` role version; `documentation-curator@0.3.0`
  becomes superseded for future jobs.
