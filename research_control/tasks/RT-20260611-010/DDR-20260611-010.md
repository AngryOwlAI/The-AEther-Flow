# Director Decision Record DDR-20260611-010

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
for a flexible source-backed HTML explainer presentation-contract migration.
The migration keeps the source-spec, source-drilldown, claim-boundary,
analysis-capsule, generated-only HTML, and structural-validation requirements
intact while adding controlled flexibility through `presentation_profile`,
`layout_intent`, and `required_content_blocks`.

## Accepted Design Decisions

- Principle: rigid authority, flexible exposition.
- New source-spec fields: `presentation_profile`, `layout_intent`, and
  `required_content_blocks`.
- `presentation_profile` is a controlled layout vocabulary documented in the
  skill and role contracts and enforced by validator constants, not a new CSV
  registry in this migration.
- `layout_intent` is free-form prose that gives the renderer page-specific
  creative direction.
- `required_content_blocks` uses page-local IDs with global syntax rules; each
  declared block must appear in generated HTML as `data-content-block="<id>"`
  and must contain source-path evidence.
- Validator scope remains deterministic and structural, not semantic or
  aesthetic.
- No full deterministic HTML generator is added in this migration.
- All tracked HTML explainer specs migrate at once.
- The project overview explainer becomes the grouped atlas index.
- README receives concise technical requirement tiers; the HTML explainer gives
  the richer explanation.
