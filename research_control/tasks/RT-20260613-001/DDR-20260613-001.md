# Director Decision Record DDR-20260613-001

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
for a GitHub-facing Markdown documentation layer under `docs/github-facing/`.

## Role-Fit Matrix

| Candidate role | Fit | Reason |
| --- | --- | --- |
| `project-control-maintainer@0.1.0` | selected | The task spans documentation files, Markdown source registration, memory discovery tooling, project-control classifier behavior, tests, generated memory/wiki outputs, and control receipts. |
| `documentation-curator@0.7.0` | rejected | The GitHub-facing pages are Documentation Curator-owned, but this transaction also changes tooling and validator behavior. |
| `memory-system-maintainer@0.1.0` | rejected | Memory tooling is part of the work, but not the full authority surface. |
| physics roles | rejected | The task explains existing project structure and must not promote scientific claims. |

## Accepted Design Decisions

- The canonical term is `GitHub-facing Markdown System`.
- The folder is `docs/github-facing/`.
- The first page set is `README.md`, `SOURCE_MANIFEST.md`, `start-here.md`,
  `project-overview.md`, `source-authority.md`, `research-system.md`,
  `claim-gates.md`, `roles-and-skills.md`, and
  `technical-requirements.md`.
- Pages are hand-authored Markdown, source-backed by existing explainer specs,
  and tracked in `SOURCE_MANIFEST.md`.
- The layer is canonical for GitHub-facing explanation and external-agent
  orientation.
- The layer is non-authoritative for physics claims and control decisions.
- Markdown source registry metadata uses role `github_facing_documentation`,
  authority status `canonical_markdown_source`, audience
  `humans_and_agents`, owner skill `documentation-curator`,
  `github_facing=true`, and `agent_documentation=true`.
- The root `README.md` links to the new GitHub-facing hub near the explainer
  guidance.

## Claim Boundary

- Allowed: add GitHub-facing explanatory Markdown, add the source manifest,
  update README front-door linkage, update glossary terminology, update
  Markdown source discovery for `docs/github-facing/`, update classifier and
  documentation-impact handling for the new lane, add tests, refresh generated
  memory/wiki surfaces, and update required control registries.
- Forbidden: canonical ontology TeX edits, manuscript TeX edits, PDF edits,
  generated wiki hand edits, generated HTML edits, physics claim promotion,
  Gate Chair verdicts, benchmark promotion, autonomous proof claims, control
  decision authority expansion, and treating GitHub-facing Markdown as
  scientific or routing authority.
