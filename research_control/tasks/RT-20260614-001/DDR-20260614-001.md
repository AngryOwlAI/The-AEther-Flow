## Accepted Design Decisions

- Root `github-facing/*.md` files are derived, source-backed reader-facing
  explainers, not body-only source-spec mirrors.
- Each GitHub-facing page declares its matching source spec, related HTML
  derivative, and `generated_noncanonical` authority status.
- Each page contains finished reader sections, source authority notes, a
  diagram reading guide, a compact external-AI navigation card, next-step
  routing, and a complete source-materials list from the matching spec.
- Source-spec Mermaid blocks are preserved byte-for-byte inside the matching
  GitHub-facing page when the source spec declares diagrams.
- `audit_documentation_surfaces.py` enforces the derived-explainer contract
  rather than exact body equality.
- Memory bootstrap discovery records GitHub-facing Markdown as
  `generated_noncanonical` and sets `related_source` / `generated_from` to the
  matching `MD-HTML-SPEC-*` object when the source spec exists.
- Tracked HTML outputs are not edited by hand in this transaction.

## Claim Boundary

Allowed: update GitHub-facing explanatory Markdown, README guidance, a
research-control design note, deterministic documentation-surface audit logic,
memory discovery metadata, focused tests, documentation-impact receipts, and
required control/documentation registries.

Forbidden: canonical ontology TeX edits, manuscript TeX edits, PDF edits,
generated HTML edits as independent authority, generated wiki hand edits,
physics claim promotion, Gate Chair verdicts, benchmark promotion, candidate
reconstruction, validator weakening, or treating GitHub-facing Markdown as
scientific or routing authority.
