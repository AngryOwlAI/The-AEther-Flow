# Director Decision Record DDR-20260612-006

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
to repair the user-reported HTML explainer reader-layer defects introduced by
the depth upgrade kit.

The objective is to remove title-section file metadata cards for
`layout_intent`, derivative authority, and registry `source_basis`; remove the
unneeded global `Expand notes`, `Collapse notes`, `Simple view`, and
`Technical view` toolbar controls; preserve useful reader features such as
reading progress, active section navigation, local search, and copyable source
chips; update reusable Documentation Curator guidance; add validator coverage;
and refresh the tracked generated HTML derivatives.

## Accepted Design Decisions

- `layout_intent` remains required source-spec metadata, but it guides rendering
  privately rather than appearing as a visible hero/title card.
- Registry `source_basis`, source-basis hash, and human-visual-only status stay
  in machine metadata and registries. Reader-facing source grounding remains in
  the Summary of subject source chips and All Source Materials section.
- The global simple/technical mode toggle is removed because its observable
  effect was only minor typography/state changes and occasional hiding of
  details.
- The global expand/collapse toolbar controls are removed because pages already
  expose the relevant details inline and the global control is not necessary
  for comprehension.
- This task removed only global toolbar buttons; RT-20260612-007 supersedes the
  earlier analysis-section allowance and removes that reader-layer contract.
- Future page-specific disclosure controls remain possible only when a source
  spec defines the need and browser QA verifies behavior.
