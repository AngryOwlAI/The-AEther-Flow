# Director Decision Record DDR-20260612-001

## Current Objective

Create and execute one bounded `project-control-maintainer@0.1.0` task overlay
for the source-backed subject-summary migration across all currently tracked
HTML explainers. The task adds a universal reader-orientation block before the
table of contents, keeps generated HTML non-authoritative, and preserves the
open exact-GR derivation boundary.

## Accepted Design Decisions

- Every tracked explainer must declare `subject_summary` as the first
  `required_content_blocks` entry and define it first under
  `## Required Content Blocks`.
- Generated HTML must render `subject_summary` as the first content block,
  immediately after the hero/title area and before the section table of
  contents.
- `subject_summary` must contain `data-summary-field` markers for
  `what_it_is`, `role_or_function`, `reader_value`, and `source_basis`.
- The `source_basis` summary field must contain visible source-path chips using
  paths declared in the source spec's `source_materials`.
- The ontology explainer adds `ontology/aether_flow_interpretation-lemen.md`
  to `source_materials` before citing it in the summary.
- Summary prose is manually authored per spec with a 120-180 word target;
  scripts validate structure and regenerate derivatives only.
- Existing analysis capsules remain separate from the summary and receive only
  narrow duplicate-opening cleanup if needed.
- The project overview hub is not exempt; its summary describes the explainer
  atlas itself.
- Documentation Curator ownership is updated through a new
  `documentation-curator@0.3.0` role version; `documentation-curator@0.2.0`
  becomes superseded for future jobs.
