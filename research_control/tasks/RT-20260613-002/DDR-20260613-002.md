## Accepted Design Decisions

- The GitHub-facing folder is root `github-facing/`.
- The folder contains body-only mirrors of the ten files under
  `markdown/html-explainer-specs/`.
- Leading YAML metadata headers are removed from the GitHub-facing mirror
  files.
- `docs/github-facing/README.md`, `docs/github-facing/SOURCE_MANIFEST.md`,
  and the compact hand-authored page set are not retained.
- The root `README.md` links each GitHub-facing Markdown file directly instead
  of linking the HTML derivatives.
- The incomplete ASCII two-track sketch is replaced with a complete Mermaid
  diagram that includes both the physics and AI research-agent tracks.
- The Markdown source registry treats root `github-facing/*.md` as
  GitHub-facing explanatory Markdown, non-authoritative for physics claims and
  control decisions.

## Claim Boundary

- Allowed: move GitHub-facing explanatory Markdown to root, mirror explainer
  spec bodies without metadata, update root README links and overview diagram,
  update glossary language, update Markdown source discovery, update classifier
  and documentation-impact handling for the new lane, add tests, refresh
  generated memory/wiki surfaces, and update required control registries.
- Forbidden: canonical ontology TeX edits, manuscript TeX edits, PDF edits,
  generated wiki hand edits, generated HTML edits, physics claim promotion,
  Gate Chair verdicts, benchmark promotion, autonomous proof claims, control
  decision authority expansion, and treating GitHub-facing Markdown as
  scientific or routing authority.
