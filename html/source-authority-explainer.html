Three-layer model

High Level, Operational, Evidence

High-Level Model

Canonical science

Registered TeX files carry scientific and derivational authority.

Canonical routing

Format-specific registries carry routing, provenance, generated-output, and memory authority.

Canonical documentation

Registered Markdown carries front-door, guidance, and source-spec authority.

Derivatives

HTML, wiki notes, PDFs, indexes, Obsidian, and .local caches are generated or scratch surfaces.

Operational Model

Edit source

Change the canonical source or registered source spec first.

Regenerate

Run bootstrap and specialized renderers to refresh derivatives.

Validate

Check source hashes, source-basis metadata, non-authority banners, and Mermaid parity.

Read carefully

Use derivatives for navigation and human understanding, not as independent authority.

Source First

Premise
The repository distinguishes source authority from generated retrieval layers.
Mechanism
Canonical files and registries own claims and provenance; derivatives expose or visualize content without becoming independent authority.
Source Basis
AGENTS.md, README.md, and source registries.
Authority Status
Human-only authority hierarchy explanation.
Uncertainty
Derivatives can become stale if hashes or metadata drift.
Validation Or Test
Run bootstrap validation and check source-basis hashes, generated-output rows, and non-authority banners.
Next Step
Edit sources or registered specs first, then regenerate.

Html Derivative

Premise
Tracked HTML helps humans understand the project but does not carry independent authority.
Mechanism
Each HTML page points to a Markdown source spec and registry source basis; Mermaid source remains in the spec and is rendered into inline SVG.
Source Basis
html-visual-explainer skill, HTML_EXPLAINER_REGISTRY.csv, and MARKDOWN_SOURCE_REGISTRY.csv.
Authority Status
Generated HTML boundary explanation.
Uncertainty
Visual polish can improve comprehension without changing authority.
Validation Or Test
Confirm source spec, metadata, controls, drilldowns, capsules, and Mermaid parity.
Next Step
Use generated pages for navigation, then inspect source specs and registries for authority.
