What This Explainer Describes

What it is

The roles-and-skills catalog is the active inventory of registered agent roles, historical role states, repo-local skills, and clearly labeled operator-context aids.

Role or function

It shows which role contracts currently govern work, which roles are superseded or gated, and which local skills provide operational procedures for continuation, memory, HTML explainers, validation, and source-first regeneration.

Why it matters first

This matters before the catalog details because skills are useful only inside the right authority boundary. The page helps readers distinguish durable project contracts, historical audit records, and convenient machine-local tools.

Source basis

The summary is grounded in the role registry, role schema, active Documentation Curator contract, and governed skill contracts that define current responsibilities and future explainer ownership. The chips separate active role authority from historical or operator-local context and current routing use.

Claim-Aware Analysis

Active Roles Are The Operating System

Premise
Readers need the current role system before historical audit context.
Mechanism
The page should visually prioritize `active` rows in `AGENT_ROLE_REGISTRY.csv`, then place `status_defined` and `superseded` roles in separate bands.
Source Basis
`registries/AGENT_ROLE_REGISTRY.csv`, role contracts under `.agents/roles/`, and `registries/ROLE_EXECUTION_REGISTRY.csv`.
Authority Status
Human-only catalog; registries and role contracts carry project-control authority.
Uncertainty
A task-local execution-role overlay can narrow or expand a role only for one AgentJob.
Validation Or Test
Check the registry row, role contract path, task-local execution-role file, and AgentJob allowlist before executing work.
Next Step
Use the role-routing explainer to inspect how a role is selected for one task.

Skills Are Repo-Local Contracts

Premise
The project can only govern skills tracked in `.codex/skills/`.
Mechanism
The catalog should list repo-local skill contracts as governed front doors, while labeling external/global tools as operator context only.
Source Basis
`.codex/skills/*/SKILL.md`, `.codex/skills/visual-explainer/subskills/mermaid-documentation/SKILL.md`, and `registries/MARKDOWN_SOURCE_REGISTRY.csv`.
Authority Status
Human-only explanation of skill contracts.
Uncertainty
A current Codex session may expose additional global skills or plugins that are useful but not durable repo requirements.
Validation Or Test
Confirm a skill path is tracked in the repository before treating it as project authority.
Next Step
Keep inferred support-skill associations explicitly labeled and source-backed.
