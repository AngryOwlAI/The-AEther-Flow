# Start Here

This page is the shortest route into the AEther-Flow repository for a GitHub reader.

Authority boundary: this page explains how to read the project. It is not authority for physics claims or control decisions.

## First Pass

1. Read [Project Overview](project-overview.md) to understand the two linked tracks: a physics research program and an AI research-agent system.
2. Read [Source Authority](source-authority.md) before relying on any claim. The repository has many readable generated surfaces, but convenience is not authority.
3. Read [Research System](research-system.md) to understand how tasks, Director decisions, AgentJobs, validators, completions, and handoffs make work auditable.
4. Read [Claim Gates](claim-gates.md) before interpreting any derivation language. Workflow progress is not scientific acceptance.
5. Use [Technical Requirements](technical-requirements.md) if you need to validate, regenerate, or inspect the system locally.

## What the Project Is

The project has two linked missions:

- A physics track: preserve an exact-GR benchmark while keeping first-principles derivation from the `AEther` / `AEther-flow` ontology open until authorized gates establish it.
- An AI research-agent track: develop a human-scaffolded theoretical-physics research workflow with explicit roles, routing, validation, claim gates, and source-first memory.

The current working goal is not to assert that GR has already been derived. The working goal is to improve the research system until it can either construct a valid derivation path or preserve reproducible obstructions.

## What to Trust

Use this order:

1. Registered `.tex` files for scientific and derivational claims.
2. Registries under [../../registries/](../../registries/) for routing, provenance, generated-output tracking, and queryable memory.
3. Registered Markdown for front-door explanation, agent guidance, control notes, and explainer source specs.
4. Generated HTML, generated wiki notes, PDFs, and `.local` surfaces as derivative aids only.
