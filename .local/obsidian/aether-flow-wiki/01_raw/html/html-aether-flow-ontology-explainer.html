Æther Flow Ontology

The ontology lane asks whether relativistic behavior can be recovered from source-defined substrate structure. The current benchmark remains exact GR. The derivation from ontology to benchmark is an open research burden, not an accepted result.

Ontology-to-Benchmark Stack

1. Substrate idea

Æther / Æther-flow

A proposed four-dimensional substrate and flow-like organization used as a research ontology. It is a candidate explanatory base, not an empirical conclusion by itself.

2. Source structures

Definitions before targets

Candidate derivation work must define source-side structures without importing the target metric, target clock behavior, or benchmark-tuned branch choices.

3. Effective behavior

Relativistic sector burden

The missing step is a coherent route from substrate data to Lorentzian metric behavior, causal structure, matter coupling, clock behavior, and invariance properties.

4. Exact-GR benchmark

Observable-scale constraint

The public benchmark preserves ordinary GR at observable scale. Any derivation proposal must recover that behavior without overclaiming.

Current Boundary

The established repository state is an exact-GR benchmark package and an open derivation program. The repository does not currently claim that the ontology has derived GR from first principles.

Derivation Burden Checklist

Lorentzian metric generation

Recover an effective metric from source data rather than assuming it.

Causal structure

Explain light cones and causal order without target imports.

Clock behavior

Account for proper-time-like behavior from source-side definitions.

Matter coupling

Recover universal coupling constraints compatible with the benchmark.

Invariance

Track coordinate and observer independence as obligations, not slogans.

Anti-smuggling

Block hidden metric, clock, or benchmark-tuned assumptions.
