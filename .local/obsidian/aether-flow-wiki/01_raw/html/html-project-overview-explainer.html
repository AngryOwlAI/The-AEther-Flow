Human-only overview

Æther-Flow Research Program

This page maps the repository’s two missions: preserve an exact-GR benchmark while testing whether a deeper Æther / Æther-flow ontology can derive it, and develop the AI research-agent system that keeps the work bounded, auditable, and source-first.

Claim boundary: this visual page explains current repository structure. It does not promote a physics result, alter a control rule, or replace registered sources.

Benchmark Exact GR is preserved at observable scale.
Open burden First-principles derivation from substrate structure remains open.
Authority model Sources and registries first; generated views second.

Two Linked Tracks

Physics research

Exact-GR benchmark plus open derivation

The physics lane keeps ordinary GR as the observable benchmark while asking whether effective Lorentzian geometry, causal structure, clock behavior, and matter coupling can be recovered from source-defined ontology data.

  • Accepted state: benchmark behavior remains ordinary GR.
  • Open state: no accepted first-principles derivation yet.
  • Guardrail: no target metric import or benchmark tuning by hand.
AI research-agent system

Bounded theoretical-physics workflow

The AI lane develops Director decisions, roles, AgentJobs, validation, registries, memory, and handoffs so proposals can be constructed, refuted, repaired, or stopped without losing scientific provenance.

  • Agent status is not physics proof.
  • Negative results are preserved as first-class research output.
  • Public release and authorship remain human-accountable.

Authority Ladder

1. Registered TeX Canonical for physics and derivational claims.
2. Registries Canonical for routing, provenance, generated outputs, and queryable memory.
3. Registered Markdown Canonical for front-door docs, agent guidance, source specs, and project notes.
4. Generated derivatives PDFs, wiki notes, indexes, Obsidian notes, and HTML explainers are useful views.
