Source Backed Html
Premise
Tracked HTML explainers are valid only when backed by registered Markdown source specs.
Mechanism
Specs declare source material, claim boundary, controls, source drilldowns, analysis capsules, and Mermaid diagrams; HTML carries metadata and marker evidence.
Source Basis
html-visual-explainer skill, visual-explainer skill, Mermaid subskill, and HTML_EXPLAINER_REGISTRY.csv.
Authority Status
Generated derivative governance explanation.
Uncertainty
Visual quality still needs human or browser review.
Validation Or Test
Run Mermaid validation, bootstrap validation, and HTML marker checks.
Next Step
Modify specs first, regenerate HTML, render inline SVG, then validate.
Documentation Impact
Premise
Documentation impact is a receipt requirement, not a role-selection shortcut.
Mechanism
The source authority class selects the role; docs-impact records cover how live changes affect future readers and agents.
Source Basis
research_control/README.md, improve-project-system skill, and validate_documentation_impact.py.
Authority Status
Control-system explanation.
Uncertainty
Resolver output is advisory; validator failures are hard stops.
Validation Or Test
Use classification, documentation-impact validation, signal validation, research-control validation, and diff checks.
Next Step
Route explanatory HTML work to Documentation Curator and validator changes to Validator Engineer.
