\begin{abstract}
This draft formalizes the Gate 0 source side for the \TheoryProgram{} without
constructing a candidate derivation. It records the minimal source primitives
already visible in the canonical ontology package, separates them from the
target exact-GR benchmark, and states the localization burden that any later
candidate must meet before it can be evaluated as more than an interpretive
restatement. The result is a control-grade science draft: it may guide the next
AgentJob, but it does not edit canonical ontology sources, promote ontology
claims, or assert a completed first-principles recovery of Einsteinian gravity.
\end{abstract}

\section{Scope and Authority}

This artifact is a Gate 0 draft under task \texttt{RT-20260608-002}. Its
authority is lower than the registered canonical ontology TeX files. It reads
the canonical foundations, dynamics, exact-closure note, and flow-geometry
manuscripts as source material, but it does not modify them.

The current canonical package fixes two facts that must remain separated.
First, the ontology posits a deeper four-dimensional substrate and intrinsic
ordered motion. Second, the active exact-closure theory adopts Einsteinian
gravity as the effective operational benchmark. Adoption is not derivation.
This Gate 0 document therefore asks only: what must be present on the source
side before a later candidate is allowed to claim that it is attempting a
derivational recovery rather than reusing the target benchmark?

\section{Source-Side Primitive Reservoir}

\begin{definition}[Substrate arena]
The source-side substrate arena is a four-dimensional differentiable object
\(\Sub\), interpreted as the deeper ontological ground of the framework. At
Gate 0, \(\Sub\) is not assumed to carry the target Lorentzian metric
\(g_{\mu\nu}\), the Einstein-Hilbert action, or the Einstein equations.
\end{definition}

\begin{definition}[Intrinsic ordered motion]
The source-side ordered motion is represented minimally by a family of maps
\[
\Phi_{\lambda}:\Sub\to\Sub,
\]
where \(\lambda\) is an ordering parameter. Gate 0 does not identify
\(\lambda\) with physical clock time and does not assume that
\(\Phi_{\lambda}\) is already generated by a GR-compatible Hamiltonian or
Lagrangian.
\end{definition}

\begin{definition}[Observer-accessible slice placeholder]
An observer-accessible slice placeholder is a map
\[
\iota_s:\Sigma_s\hookrightarrow\Sub,
\]
where \(\Sigma_s\) is a three-dimensional slice labelable by an order
parameter \(s\). This placeholder records the intended direction of
explanation: observed three-dimensional structure is to be localized from the
substrate side rather than imported as an already-relativistic spacelike
hypersurface.
\end{definition}

\begin{definition}[S-time placeholder]
S-time is represented at Gate 0 by an order functional
\[
S=\Order[\Gamma,\Psi,\Sub,\Phi_{\lambda}],
\]
defined schematically on an observer history \(\Gamma\), physical state
\(\Psi\), substrate arena \(\Sub\), and ordered motion
\(\Phi_{\lambda}\). A future candidate may refine this object, but it may not
replace it by ordinary proper time without deriving the replacement.
\end{definition}

These four objects form a minimal reservoir. They are not yet a theory of
gravity. They are the source-side vocabulary that a later candidate must turn
into a controlled bridge.

\section{Target Benchmark Kept Out of the Source}

The exact-closure package fixes the target benchmark:
\[
S_{\mathrm{eff}}
=
\frac{c^3}{16\pi G}\int d^4x\,\sqrt{-g}\,R
+
S_{\mathrm{matter}}[g,\psi],
\]
with the Einstein equations, null structure, proper-time structure, and
ordinary matter coupling interpreted in the standard way. Gate 0 permits this
benchmark only as a recovery target, comparison object, or failure test.

\begin{constraint}[No target import]
A candidate source model fails Gate 0 if it assumes any of the following as
source primitives:
\begin{enumerate}
    \item a Lorentzian metric \(g_{\mu\nu}\) with the target causal structure;
    \item the Einstein-Hilbert action or Einstein tensor;
    \item universal matter coupling to a metric already identified with the
    operational spacetime metric;
    \item a GR observer congruence \(u^\mu\) used as if it were the substrate
    ordering law;
    \item diffeomorphism gauge structure copied from the target rather than
    recovered or justified from source-side redundancy.
\end{enumerate}
\end{constraint}

The flow-geometry congruence language remains useful as an exact-closure
dictionary. It is not yet a source-side derivation. A later candidate may use a
congruence as an intermediate object only after specifying how the substrate
variables select or induce it.

\section{Gate 0 Localization Burden}

\begin{burden}[Source localization data]
Before candidate construction begins, the source packet should identify the
data
\[
\mathcal{D}_{0}
=
(\Sub,\Phi_{\lambda},\chi,\mathcal{L},\mathcal{C}),
\]
where \(\chi\) denotes additional source variables if any,
\(\mathcal{L}\) is the proposed localization map from source structure to
observer-accessible structure, and \(\mathcal{C}\) is a list of constraints
or symmetries that prevent target smuggling.
\end{burden}

\begin{burden}[Observer localization]
The localization map \(\mathcal{L}\) must explain how embedded observers,
spatial slices, clock order, and lightlike propagation become meaningful from
the source side. It is insufficient to state that observers experience a
metric unless the map specifies what source-side condition makes that metric
available.
\end{burden}

\begin{burden}[Dimensional and limiting consistency]
The source packet must state dimensions and limiting regimes. In particular,
it must say which source quantities carry units, which are dimensionless,
which limiting operation is intended to produce the effective relativistic
regime, and which variables remain unobservable gauge or redundancy data.
\end{burden}

\begin{burden}[Recovery target]
The target is exact recovery of the benchmark effective sector in the
appropriate limit:
\[
(\Sub,\Phi_{\lambda},\chi,\mathcal{L},\mathcal{C})
\quad\Longrightarrow_{\mathrm{candidate}}\quad
(M,g_{\mu\nu},S_{\mathrm{eff}},\psi)
\]
without placing \(g_{\mu\nu}\) or \(S_{\mathrm{eff}}\) in the initial source
data. The arrow above is a burden marker, not a result.
\end{burden}

\section{Allowed and Forbidden Next Moves}

Allowed next moves:
\begin{itemize}
    \item select one narrow source packet for Candidate Constructor review;
    \item define additional source variables \(\chi\) if they are not copied
    from the target metric sector;
    \item propose a localization map \(\mathcal{L}\) with explicit
    assumptions and failure modes;
    \item prepare a smuggling-audit checklist before any derivational claim is
    made.
\end{itemize}

Forbidden next moves:
\begin{itemize}
    \item editing canonical ontology TeX from this draft;
    \item treating the exact-closure benchmark as if it were a source-side
    primitive;
    \item claiming that Gate 0 has produced a completed derivation;
    \item moving directly to Gate Chair without a candidate, refutation pass,
    and smuggling audit.
\end{itemize}

\section{Result}

The Gate 0 source side is now formalized as a burden ledger rather than a
candidate derivation. The source reservoir consists of
\(\Sub\), \(\Phi_{\lambda}\), \(\iota_s\), and \(S\), with optional future
variables \(\chi\). The key missing object is the localization map
\(\mathcal{L}\). The logical next scientific role, if the Director agrees in a
separate continuation, is therefore Candidate Constructor with a narrow mandate
to propose \(\mathcal{D}_{0}\) or to report that no non-smuggled localization
packet is available.
